# 프로젝트 운영 프로토콜

이 문서는 AI 어시스턴트(안티그래비티)가 이 프로젝트를 수행할 때 지켜야 할 특별 규칙을 정의합니다.

## 1. 기본 운영 규칙
- **명령 중심**: 항상 사용자님이 명령한 내용만 수행하고 즉시 중단합니다.
- **보고 필수**: 작업이 끝나면 항상 "작업 완료했습니다."라고 상세히 보고합니다.
- **사전 승인**: 명령 이외의 추가 작업이 필요하다고 판단될 경우, 반드시 그 이유와 중요성을 설명하고 허락을 구한 뒤 진행합니다.

## 2. 작업 종료 프로토콜

### 명령: "오늘 작업 종료하자"
이 명령이 내려지면 다음 두 단계를 자동으로 수행합니다:
1. **프롬프트 저장**: 해당 세션의 모든 요청 사항을 `프롬프트-개별-시간순.md` 파일에 업데이트합니다.
2. **깃허브 업로드**: 로컬의 모든 변경 사항을 `git push`를 통해 원격 저장소에 동기화합니다.

### 명령: "오늘 작업 종료하자. 전원도 꺼줘."
위의 두 단계(저장 및 업로드)를 모두 마친 후, **시스템 전원을 완전히 종료(Shutdown)**합니다.
